最简单的一个函数式编程组件（相比起类组件的方式，函数式组件更加简洁简单）

```tsx
import { useState } from 'react'
import './App.css'

function App() {
  const [count, setCount] = useState(0)

  return (
    <div onClick={() => setCount(count + 1)}>
    {count}
    </div>
  )
}

export default App
// 1. 简洁代码量少
// 2. 消除了Class组件中this关键字带来的困扰
// 3. Hooks的设计让状态逻辑的复用变得异常简单，可以轻松将逻辑封装到自定义Hook中
// 4. 现代化React开发标准：函数式组件+Hooks
```

### 3.1. 组件化开发核心

一个 React 组件，本质上是：**输入（Props） → 内部状态（State） → 渲染 UI → 响应交互（事件） → 状态更新 → 重新渲染**

可以理解为一个**受控的状态机 + 渲染函数**

####  `Props`  

1. 组件的参数，由父组件传入，只读不可变。
2. 特点：单向数据流，不允许组件内部修改，用于组件复用和解耦
3. 本质：函数参数；``React`组件本质就是函数

```tsx
// 子组件
import type React from "react";
import { useState } from "react";

interface HelloWorldProps { // 事先声明好props的类型
    title: string;
    // render相当于一个插槽 render props，还可以从内部组件传参数出去
    render?: (count: number) => React.ReactNode 
}

export function HelloWord(props: HelloWorldProps) {
    const { title, render } = props // 可以直接解构
    const [count, setCount] = useState(0) // 声明一个变量，给子组件使用，也可以给父亲组件使用
    // 单节点
    return (
        <>
            <button onClick={() => setCount(count + 1)}>button</button>
            <div>{title} -- {render?.(count)}</div> // render属性的使用就是直接函数调用即可。
        </>
    )
}
```
