    # 2. ref 与 props

    # 2.1 ref

    1. 被用来该元素或子组件注册引用信息（id的替代者）
    2. 应用在html标签上的获取真实dom元素，应用在组件标签上是组件实例对象【vc】。
    3. 使用方式

        <button @click="showDom">点击获取</button>
    </div>
</template>
<script>
    export default({
        name:'App',
        methods:{
            showDom(){
                console.log(document.getElementById("h1"));
            }
        }
    })

	<div>

        components:{

                console.log(this.$ref.btn);          // 同上

    # 2.2 props 配置

    第一种：只接收数据。这样子传过去的age不是number类型，而是string类型。

    <Students name="tom" age="18" sex="男" />
    1
    props:["name","sex","age"]
    1

    在age前加 : 可以传过去number类型的age，即v-bind:age="18"是把引号里的值 18 传过去，而不是字符串"18"。方便对age进行加减。

    <Students name="tom" :age="18" sex="男" />
    1

    第二种：手动进行类型限制。使用对象声明props

    props:{
    age:Number,
    name:String,
    sex:String
}
    1
    2
    3
    4
    5

    在传递时同样是string类型，但是因为做了类型限制，所以会在终端有提示报错，age前应该加上 : ，传递number类型。

    第三种：类型限制+默认值+必需属性

    props:{
    name: {type: String, required: true, default:xxx}
    // age sex 不详细
}
    1
    2
    3
    4

    一般require和default不同时出现。毕竟必传的值也不需要默认值。

    👂 props是只读的，vue底层会监测你对props的修改否则会发出警告。不能对外部传入的props属性进行修改。可以定义data属性指向props的属性，再对其进行修改。

    👂 props的属性优先级大于data的数据的优先级。

    # 3. mixin 混入

    # 3.1 理解

    🖐 参考链接 彻底搞懂Vue中的Mixin混入（保姆级教程） - 掘金 (juejin.cn) (opens new window)

    将组件的 公共逻辑或者配置 提取出来，哪个组件需要用到时，直接将提取的这部分混入到组件内部即可。这样既可以减少代码冗余度，也可以让后期维护起来更加容易。

    👊即：mixin是组件中的组件，Vue组件化提高代码复用性。组件与组件之间还有重复部分，使用mixin再抽离一遍。

    功能：可以把多个组件共用的配置提取成一个混入对象。分发 Vue 组件中的可复用功能。

    使用方式：

    1. 定义混合在单独的js文件里面：对象
    // src/mixin/index.js
export const mixins = {
  data() {
    return {};
  },
  computed: {},
  created() {},
  mounted() {},
  methods: {},
};
    1
    2
    3
    4

    1. 局部混入
    import {mixins} from './mixin/index';
export default {
    name:'App',
    mixins:[mixins]    // 直接使用
}
    1
    2
    3
    4
    5

    mixin中的生命周期函数会和组件的生命周期函数一起使用执行。

    mixin中的data数据在组件中也可以使用。

    mixin中的method在组件内部也可以使用。

    执行顺序：mixin==> 组件

    不同组件中的mixin是相互独立的！

    1. 全局混入
    // main.js中
import {mixins} from './mixin/index';
Vue,mixin(mixins);
    1
    2
    3

    即可以在所有组件中进行使用。

    1. 发生冲突
    • 当mixin中的data数据与组件中的data数据冲突时，组件中的data数据会覆盖mixin中数据。
    • 同理在method方法一致时也是调用组件的方法。

    # 3.2 与vuex区别

    • Vuex公共状态管理，如果在一个组件中更改了Vuex中的某个数据，那么其它所有引用了Vuex中该数据的组件也会跟着变化。
    • Mixin中的数据和方法都是独立的，组件之间使用后是互相不影响的。

    # 5. webStorage

    • 浏览器本地存储
    • 存储内容大小一般在 5MB 左右。cookie是4KB

    # 5.1 localStorage

    1. 存在window身上。访问时是window.localStorage

    2. 方法：

      1. setItem(key,value)：如果存储的不是字符串，那么就会主动强制转换为字符串。调用toString

        如果是对象，就会被转为 [object Object]，无法知道里面具体的内容。所以我们在存储对象的值时，需要手动进行字符串转换，调用Json.stringify

      2. getItem(key)：当然读取到对象的时候还是字符串就需要使用 Json.parse进行解析。读不到的就是null

      3. removeItem(key)：把键名从存储中删除。

      4. clear()：清空存储中的所有数据。

    3. 把浏览器关闭，localStorage的记录也不会消失。需要用户手动清空。

    # 5.2 sessionStorage

    1. 同localStorage的所有API
    2. 浏览器窗口关闭，记录也会随之消失。
    3. Json.parse(null) == null
