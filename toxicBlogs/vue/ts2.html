    # 1. 面向对象OOD

    # 概念

    👊 : 1. 什么是对象？

    📃 : 任何事物到了程序中就变成了一个对象。程序是对事物的抽象。

    1. 操作浏览器要使用window对象
    2. 操作网页要使用document对象
    3. 操作控制台要使用console对象

    一切操作都要通过对象。

    👊 : 2. 属性与方法

    📃 : 在程序中所有的对象都被分成了两个部分数据和功能。

    ​ 以人为例，人的姓名、性别、年龄、身高、体重等属于数据，人可以说话、走路、吃饭、睡觉这些属于人的功能。

    ​ 数据在对象中被成为属性，而功能就被称为方法。所以简而言之，在程序中一切皆是对象。

    # 2. 类

    **类是对象的抽象，对象是类的实例。**要创建对象，必须先定义类。可以把类理解为对象的模型。

    # 2.3 构造函数

    👊 一个类可能创建多个对象。希望这个类用来创建不同的对象，而不是一模一样的对象。

    // 使用class关键字定义一个类
class Dog {
    name: string;
    age: number;

    bark() {
        alert("汪汪汪");
    };

    // 调用new关键字就是执行构造函数。
    constructor(name:string,age:number) {
        // 构造函数中有this关键字。就表示当前实例，我们可以通过this向新建的对象中添加属性。
        this.name = name;
        this.age = age;
    }
}

const dog = new Dog("旺财",2);
console.log(dog);

    ⁉️ 子类写构造函数时，会覆盖掉父类的构造函数。所以必须调用父类的构造函数。

    class Dog extends Animal {

    sex:string;

        super(name,age);  // 1. 父类的构造函数中有属性name和age。所以需要在子类的构造函数也声明。
        // 3. 再添加上子类需要的属性。

    # 4. 接口

    # 4.1 抽象类

    📃 按道理来说：父类如Animal也是一个类，也可以用来创建对象。但是我们不希望它被用来创建对象，它是作为一个超类被继承。

    👊 在超类的前面加上**abstact**关键字将它变为抽象类。抽象类不能被用来创建对象。

    👊 **抽象类就是专门用来继承的类。**抽象类中可以添加抽象方法【也是在函数前面加上关键字abstact，且没有方法体】。具体实现必须由子类实行。

    // 使用class关键字定义一个类
abstract class Animal {
    name: string;

    // 调用new关键字就是执行构造函数。
    constructor(name: string) {
        // 构造函数中有this关键字。就表示当前实例，我们可以通过this向新建的对象中添加属性。
        this.name = name;
    };

    abstract bark(): void;
}

    👊 接口可以重复声明。不会覆盖，会添加上去。

    👊 接口中所有方法都是抽象方法，所有属性都没有具体值。

    // 有点像抽象类。
// 使用关键字定义接口
interface MyInterface{
    // 接口属性声明
    name:string;
    // 接口方法是抽象方法，要写返回值，不写方法体
    bark():void;
}

// 抽象类也是类。需要class
abstract class Animal{
    // 声明属性
    name:string;
    // 声明属性之后需要写构造函数。
    protected constructor(name:string) {
        this.name = name;
    }
    // 只要加上abstract关键字变为抽象方法，才可以不写方法体。
    // 一般的方法还是要写的，但接口中天然是抽象方法。
    abstract bark():void;
}

    # 5. 泛型

    # 5.1 属性封装

    👊 现在属性是在对象中设置的。属性可以通过对象被任意地修改。导致对象中的数据变得非常不安全。

    class MyClass implements MyInterface{
    name: string;
    constructor(name:string) {
        this.name = name;
    }
}
const dog = new MyClass("旺财");
dog.name = "喵咪咪";
console.log(dog); // 输出 Object { name: "喵咪咪" }

    # 5.2 泛型

    👊 在定义函数或者类时，如果类型不明确就可以使用泛型。

    function fn<T>(a:T):T{
    // fn后的<T>是定义T类型。形参为T类型，返回类型也为T类型
    // T类型是尚不明确的类型。
    return a;
}
// 调用该函数时，会自动判断参数的类型。
fn(10);
fn<string>("hello");
