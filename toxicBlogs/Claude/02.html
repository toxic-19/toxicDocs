    # 1. JavaScript 迭代器协议

    JavaScript 中有两种迭代器协议。

    1. 同步迭代器：Symbol.iterator
    const arr = [1, 2, 3];
const iterator = arr[Symbol.iterator]();
iterator.next(); // { value: 1, done: false }
iterator.next(); // { value: 2, done: false }
iterator.next(); // { value: 3, done: false }
iterator.next(); // { value: undefined, done: true }

    很显然同步迭代器是 next() 直接返回结果。

    实现方法：

    // 一个同步可迭代对象
const syncIterator = {
    data: [1, 2, 3],
    index: 0,
    [Symbol.iterator]() {
        return this;
    },
    next() {
        if (this.index < this.data.length) {
            return { value: this.data[this.index++], done: false };
        }
        return { value: undefined, done: true };
    }
};
// 使用 for...of 遍历
for (const item of syncIterator) {
    console.log(item); // 1, 2, 3
}

    1. 异步迭代器：Symbol.asyncIterator
    // 一个异步可迭代对象（数据源是异步的，比如从网络获取）
const asyncIterator = {
    data: [1, 2, 3],
    index: 0,
    [Symbol.asyncIterator]() {
        return this;
    },
    // 关键区别：next() 返回 Promise                                                                           
    next() {
        return new Promise((resolve) => {
            setTimeout(() => {
                if (this.index < this.data.length) {
                    resolve({ value: this.data[this.index++], done: false });
                } else {
                    resolve({ value: undefined, done: true });
                }
            }, 1000); // 模拟异步延迟
        });
    }
};

async function main(asyncIterator: any) {
    // 使用 for await...of 遍历                                                                              
    for await (const item of asyncIterator) {
        console.log(item); // 1, 2, 3（每秒一个）                                                               
    }
}
main(asyncIterator);

    异步迭代器的 next() 是直接返回一个Promise。

    为什么需要异步迭代器：需要等待上一个操作结束后，才执行第二个。

    因为 asyncIterator.next() 是异步，所以在 for … of中需要加上 await

    # 2. readline

    在处理learn-claude-code时，需要获取用户的输入，与模型对话后，输出模型结果再继续输出提示语如 S01>>

    内部执行顺序：

    1. 调用 rl[Symbol.asyncIterator]().next() → 返回一个 Promise（等待 line 事件）
    2. 用户输入 hello —> line 事件触发 —> promise reslove
    3. await 得到值之后 处理 hello 此时for循环暂停
    4. 处理完后才调用 next 到下一行
    // 模拟 readline 的核心逻辑
class SimpleReadline { 
  constructor() {
    this.queue = []; 
    this.resolvers = [];
  }

  // 模拟用户输入（正常情况下是 stdin 触发
  emitLine(input) {
    if (this.resolvers.length > 0) { 
      const resolve = this.resolvers.shift(); 
      resolve({ value: input, done: false });
    } else {              
      this.queue.push(input);
    }                 
  }

  [Symbol.asyncIterator]() {
    const self = this;
    return {
      next() { // 必须返回的next()对象
        // 如果已有排队的输入，直接返回
        if (self.queue.length > 0) { 
          const input = self.queue.shift();
          return Promise.resolve({ value: input, done: false }); 
        }              
        // 否则等待新的输入   
        return new Promise((resolve) => {
          self.resolvers.push(resolve);
        });
      }
    };    
  }
}

// 测试
async function main() {
  const rl = new SimpleReadline();

  // 模拟用户分3次输入   
  setTimeout(() => rl.emitLine("hello"), 100);
  setTimeout(() => rl.emitLine("world"), 500);                                                            
  setTimeout(() => rl.emitLine("exit"), 900);
  for await (const input of rl) {
    console.log(`用户输入: ${input}`);       
    if (input === "exit") break;
  }      
  console.log("程序结束");
}               
main();
// 输出:
// 用户输入: hello
// 用户输入: world
// 用户输入: exit
// 程序结束
