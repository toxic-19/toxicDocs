    # 1. 本文档学习目标

    1. 系统掌握React19的核心概念与组件化开发思想，并能熟练运用TypeScript进行类型约束。

    2. 深入理解并实战React19的革命性新特性，包括但不限于Actions、ReactCompiler（概念与影响）、useHook、以及服务端组件的基础。

    3. 熟练掌握现代React技术栈，包括状态管理（Zustand/ReduxToolkit）、路由（ReactRouter）、样式方案（Tailwind CSS/CSS-in-JS）和数据请求（ReactQuery/SWR)。

    4. 具备独立构建、测试和部署一个完整的、可维护、高性能的全栈React应用的能力。

    5. 掌握React性能优化的核心思路与实践，从传统的手动优化（useMemo，useCallback）到理解ReactCompiler的自动优化机制。

    6. 深入理解React的核心工作原理，如虚拟DOM、Fiber架构和并发渲染，能够从容应对深度面试和解决复杂问题。

    # 2. 思想演进

    1. Jquery+HTML：直接操作DOM，导致UI状态与数据状态的同步变得困难。

    2. MVC+MVVM：AngualrJS+早期Vue，通过数据绑定的形式将开发者从繁琐的DOM操作中解放出来

    3. React：用户界面仅仅是应用程序状态的一个函数即 UI = f(state)

      只需要清晰的描述出在任何特定状态下，界面应该是什么样子。React会以最优的方式计算出新旧界面之间的差异，并更新真实DOM中需要变化的部分。

      被称为：声明式编程。状态驱动视图更新

    # 3.1. 组件化开发核心

    一个 React 组件，本质上是：输入（Props） → 内部状态（State） → 渲染 UI → 响应交互（事件） → 状态更新 → 重新渲染

    可以理解为一个受控的状态机 + 渲染函数

    # Props

    1. 组件的参数，由父组件传入，只读不可变。
    2. 特点：单向数据流，不允许组件内部修改，用于组件复用和解耦
    3. 本质：函数参数；React组件本质就是函数
    // 子组件
import type React from "react";
import { useState } from "react";

interface HelloWorldProps { // 事先声明好props的类型，即时反馈机制，TS编译器和代码编辑器可以立刻给出错误提示
    title: string;
    // render相当于一个插槽 render props，还可以从内部组件传参数出去
    render?: (count: number) => React.ReactNode;
    onChange?: (count: number) => void; // 希望由父组件传递一个onChange函数，定义具体的操作，而本组件只需要知道什么时候触发和传递即可，如果函数中定义了参数，说明需要进行参数传递，否则就只是行为触发。
}

export function HelloWord(props: HelloWorldProps) {
    const { title, render, onChange } = props // 可以直接解构
    const [count, setCount] = useState(0) // 声明一个变量，给子组件使用，也可以给父亲组件使用
   	const handleAdd = () => {
      setCount(count + 1) // setCount是异步更新，所以说接下来的onChange中传递的还是count+1
      onChange?.(count + 1)
    }
    // 单节点
    return (
        // onClick后为事件处理函数
        <>
            <button onClick={handleAdd}>button</button> // 如果直接写内联的函数的话必须为箭头函数
            <div>{title} -- {render?.(count)}</div> // render属性的使用就是直接函数调用即可。
        </>
    )
}

    # 事件处理与合成事件系统

    数据通过 Props 从父组件流向子组件。

    子组件要改变父组件的状态，这种自下而上的通信，需要通过事件处理的实现。如 onClick 驼峰式命名。传递函数作为事件处理程序。

    核心思想就是将父组件中的函数作为Prop传递给子组件。

    父组件定义了行为（做什么），子组件则决定了何时触发该行为（何时调用该函数）。

    示例可配合上面的代码块一起查看：

    import { HelloWord } from './components/HelloWorld'

function App() {
  return (
    <HelloWord
      onChange={() => console.log('父组件接收到子组件的通信')} // 注意onClick等事件后都是箭头函数
    />
  )
}

    # 状态管理

    Props是从外部传入且不可变的，而State则是组件内部自己管理的数据，并且它是可变的。

    在函数式组件中，我们用来赋予组件状态能力的工具，就是 React 最基础也最重要的 Hook 之一：useState。

    useState的调用本身非常简单，它接收一个参数作为状态的初始值，然后返回一个包含两个元素的数组。

    我们通常使用JavaScript的数组解构语法来接收这两个值：

    const [count, setCount] = setState(0) // 数据 - 如何去修改它的数据（异步更新）
const [user, setUser] = setState({ name: 'Tom', age: 19 }) // 注意修改要整个对象修改。

    对于对象或数组这样的引用类型，我们不应该直接修改它们内部的属性，而应该创建一个新的对象或数组来替换l旧的。

    这是因为React通过浅比较来判断状态是否发生了变化。如果你只是修改了原对象的属性，对象的引用地址并未改变，React可能会认为状态没有变化，从而跳过重新渲染。

    # 列表渲染

    1. 核心方式：map() 本质上通过数组转为JSX数组由React自动渲染
    2. key的作用：用于 标识节点唯一性（diff 算法） 提高性能+避免错位更新
    import { useState } from "react"

export function List() {
    const [list, setList] = useState<number[]>([]) // 声明类型，避免list.length一直报错

    return (
        <div>
            {/* 代码块js+html */}
            {list.map((item) => (
                // key是React内部使用的，不会传递给组件
                <div key={item}>{item}</div> // 不使用index是因为列表的变动（插入、删除、排序）等会导致DOM复用错位
            ))}
            <button onClick={() => setList([...list, list.length])}>追加元素</button>
        </div>
    )
}

    # key 的重要性

    key是React 用来识别列表中各个元素的“身份证”。

    当列表数据发生变化（例如增、删、改、排序）时，React 的协调算法（Reconciliation）会通过key来高效地比对新旧两棵虚拟DOM树。它会根据key来判断哪些元素是新创建的、哪些被删除了、哪些只是移动了位置。

    1. 一个稳定且唯一的key 能帮助 React 最大限度地复用已有的 DOM 元素和组件实例，从而极大地提升性能。

    2. 如果不提供 key，React 会在控制台给出警告，并且在列表更新时可能会出现不可预测的 UI bug 和性能问题。

    3. 使用数组的索引（index）作为key 是一种常见的反模式，应当极力避免。因为当列表项的顺序发生改变时（例如在数组开头插入一个新元素），所有后续元素的索引都会改变。这会让React误以为是元素自身的内容发生了大规模变化，从而导致不必要的重新渲染，甚至丢失组件内部的状态（如输入框的内容）。
